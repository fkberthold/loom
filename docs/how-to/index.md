# How-to guides

How-to guides are the *application + action* quadrant: focused recipes
for the already-competent user who knows what they want to do. Each
guide solves one problem; titles state the task; steps are
executable.

> **What how-to guides are not.** They are not tutorials (no
> teaching, no narrative arc) and not reference (no surface
> catalogue). If you're learning loom for the first time, start at
> [Tutorials](../tutorials/index.md) instead.

## What's here

- [**Install loom**](install.md) — symlink-based install into
  `~/.claude/`, with backup of any pre-existing files loom owns.

More how-to pages land in subsequent restructure beads (loom-9z1.4):
configuring hooks, writing a new skill, troubleshooting stuck
workflows, and switching workflow modes per project.
